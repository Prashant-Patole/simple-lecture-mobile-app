AI Support Reply System & "Need Human" Functionality - Complete Documentation
Overview
The support system follows an AI-first approach where:

User creates a support ticket
AI attempts to resolve the issue
User provides feedback (Resolved / Need Human Support)
If escalated, Admin manually responds
1. Database Schema
support_tickets Table
Column	Type	Description
id	uuid	Primary key
user_id	uuid	The user who created the ticket
category	text	Ticket category (account, payment, technical, etc.)
subject	text	Short description of the issue
status	text	Current status (open, ai_responded, escalated_to_admin, admin_responded, closed_resolved)
ai_confidence	numeric	AI's confidence score (0-1)
escalated_at	timestamptz	When ticket was escalated
assigned_admin_id	uuid	Admin assigned to handle
created_at	timestamptz	Creation timestamp
updated_at	timestamptz	Last update
closed_at	timestamptz	When ticket was closed
support_messages Table
Column	Type	Description
id	uuid	Primary key
ticket_id	uuid	Foreign key to support_tickets
sender_type	text	Who sent: user, ai, or admin
content	text	Message content
created_at	timestamptz	When message was sent
2. AI Reply Flow
Step 1: User Creates Ticket

// src/hooks/useSupportAssistant.ts - createTicket function
const createTicket = async (category: string, subject: string, initialMessage: string) => {
  // 1. Create ticket in database
  const { data: ticket } = await supabase
    .from('support_tickets')
    .insert({
      user_id: user.id,
      category,
      subject,
      status: 'open'  // Initial status
    })
    .select()
    .single();

  // 2. Save the initial message
  await supabase.from('support_messages').insert({
    ticket_id: ticket.id,
    sender_type: 'user',
    content: initialMessage
  });

  return ticket;
};

4. "Need Human Support" (Escalation) Flow
User Clicks "Need Human Support":

// src/hooks/useSupportAssistant.ts - escalateTicket function
const escalateTicket = async (ticketId: string) => {
  const { error } = await supabase
    .from('support_tickets')
    .update({ 
      status: 'escalated_to_admin',        // Change status
      escalated_at: new Date().toISOString()  // Record escalation time
    })
    .eq('id', ticketId);

  if (error) {
    toast({ title: "Failed to escalate ticket", variant: "destructive" });
    return false;
  }

  toast({ title: "Ticket escalated to support team. They will respond shortly." });
  return true;
};
Handle Escalation in UI:

// src/pages/Support.tsx
const handleEscalate = async () => {
  if (currentTicketId) {
    const success = await escalateTicket(currentTicketId);
    if (success) {
      setShowChat(false);   // Close chat
      clearMessages();       // Clear conversation
    }
  }
};
5. "Resolved" Flow

// src/hooks/useSupportAssistant.ts - resolveTicket function
const resolveTicket = async (ticketId: string) => {
  const { error } = await supabase
    .from('support_tickets')
    .update({ 
      status: 'closed_resolved',           // Mark as resolved
      closed_at: new Date().toISOString()  // Record close time
    })
    .eq('id', ticketId);

  if (error) {
    toast({ title: "Failed to close ticket", variant: "destructive" });
    return false;
  }

  toast({ title: "Ticket resolved successfully!" });
  return true;
};

7. Complete Ticket Status Flow
OPEN
  ↓ (User sends message)
AI_RESPONDED
  ↓ (User clicks feedback button)
  ├─→ "Yes, Resolved" → CLOSED_RESOLVED
  └─→ "Need Human Support" → ESCALATED_TO_ADMIN
                                    ↓ (Admin replies)
                              ADMIN_RESPONDED
                                    ↓ (Admin closes)
                              CLOSED_RESOLVED
8. API Endpoints Summary
Endpoint	Method	Purpose
/rest/v1/support_tickets	POST	Create new ticket
/rest/v1/support_tickets?id=eq.{id}	PATCH	Update ticket status
/rest/v1/support_messages	POST	Save message (user/ai/admin)
/rest/v1/support_messages?ticket_id=eq.{id}	GET	Fetch ticket messages
/functions/v1/ai-support-assistant	POST	Get AI response (streaming)
