Complete Documentation: Mock & PYQs Tab
Overview
The Mock & PYQs Tab is a comprehensive test-taking system that allows students to practice with:

Previous Year Questions (PYQs) - Past exam papers
Proficiency Tests - Skill assessment tests
Mock Exams - Full-length practice exams
1. Database Tables
Table: subject_previous_year_papers
Stores paper/test metadata.

Column	Type	Nullable	Description
id	uuid	NO	Primary key
subject_id	uuid	NO	Foreign key to subjects
year	integer	NO	Year of the paper (e.g., 2024)
exam_name	text	NO	Name like "JEE Main", "NEET"
paper_type	text	YES	"Shift 1", "Morning", etc.
pdf_url	text	YES	Link to download PDF
total_questions	integer	YES	Number of questions
chapter_id	uuid	YES	Optional chapter filter
topic_id	uuid	YES	Optional topic filter
document_type	text	YES	mcq, practice, proficiency
paper_category	text	YES	previous_year, proficiency, exam
created_at	timestamp	YES	Creation timestamp
Table: questions (for paper questions)
Questions linked to papers via previous_year_paper_id.

Column	Type	Description
id	uuid	Primary key
previous_year_paper_id	uuid	Links to paper
question_text	text	Question content (supports LaTeX)
question_type	text	mcq, subjective, true_false, integer
question_format	text	single_choice, multiple_choice, subjective, integer
options	jsonb	{"A": {"text": "..."}, "B": {"text": "..."}}
correct_answer	text	The correct answer
explanation	text	Solution explanation
difficulty	text	Easy, Medium, Hard
marks	integer	Marks for the question
is_important	boolean	Mark for important questions
Table: paper_test_results
Stores student test submissions.

Column	Type	Description
id	uuid	Primary key
student_id	uuid	User ID
paper_id	uuid	Paper ID
subject_id	uuid	Subject ID
paper_category	text	previous_year, proficiency, exam
score	integer	Correct answers count
total_questions	integer	Total questions attempted
percentage	numeric	Score percentage
time_taken_seconds	integer	Duration in seconds
answers	jsonb	{"question_id": "user_answer", ...}
grading_status	text	pending, graded, ai_graded
submitted_at	timestamp	Submission time
graded_at	timestamp	When AI grading completed
Table: student_answers
Stores individual written answers with images.

Column	Type	Description
id	uuid	Primary key
user_id	uuid	Student ID
question_id	uuid	Question ID
paper_id	uuid	Paper ID
answer_text	text	Text answer
answer_image_url	text	Uploaded image URL
extracted_answer_text	text	AI-extracted text from image
extraction_confidence	text	high, medium, low
submitted_at	timestamp	Submission time
2. API Endpoints (Supabase REST)
Get Papers for a Subject

GET /rest/v1/subject_previous_year_papers?subject_id=eq.{subject_id}&order=year.desc
Query Parameters:

subject_id=eq.{uuid} - Filter by subject (required)
chapter_id=eq.{uuid} - Filter by chapter (optional)
topic_id=eq.{uuid} - Filter by topic (optional)
topic_id=is.null - Get chapter-level papers only

3. Edge Functions (AI-Powered)
Extract Answer from Image

POST /functions/v1/extract-answer-from-image
Request:


{
  "image_url": "https://storage.url/answer.jpg",
  "question_context": "What is the integral of sin(x)?"
}
Response:


{
  "extracted_text": "2",
  "confidence": "high"
}
Confidence Levels:

high - Clear, short answer extracted
medium - Answer extracted with some uncertainty
low - Image unreadable or no text found
AI Compare Math Answers

POST /functions/v1/ai-compare-math-answers
Used for grading integer/numerical answers where notation may vary.

Request:


{
  "items": [
    {
      "id": "q1-uuid",
      "user_answer": "5^2",
      "correct_answer": "25"
    },
    {
      "id": "q2-uuid",
      "user_answer": "\\frac{1}{2}",
      "correct_answer": "0.5"
    }
  ]
}
Response:


{
  "results": [
    { "id": "q1-uuid", "is_equivalent": true },
    { "id": "q2-uuid", "is_equivalent": true }
  ]
}
4. Question Types Supported
Type	Format	Answer Method	Grading
mcq	single_choice	RadioGroup (A/B/C/D)	Automatic
mcq	multiple_choice	Checkbox (multiple)	Automatic
true_false	-	RadioGroup (True/False)	Automatic
integer	integer	Text input (number)	Automatic + AI fallback
subjective	subjective	Textarea + image upload	AI extraction from image
5. Test Flow States
papers → setup → testing → results
   ↑         ↑         ↓
   └─────────┴─────────┘
State	Description
papers	Paper selection grid with categories
setup	Configure question count and time limit
testing	Active test with timer and navigation
results	Score display and review (saved to DB)

8. Answer Normalization
The system includes sophisticated answer normalization for mathematical equivalence:

Handles:

LaTeX delimiters: $...$, $$...$$, \(...\), \[...\]
Superscripts: ² ↔ ^2
Subscripts: ₁ ↔ _1
Unicode math: × ↔ *, ÷ ↔ /, − ↔ -
Fractions: \frac{1}{2} ↔ (1)/(2)
Greek letters: π ↔ PI
