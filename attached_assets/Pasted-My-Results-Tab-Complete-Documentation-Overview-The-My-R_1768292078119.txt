My Results Tab - Complete Documentation
Overview
The "My Results" tab displays all test submissions from a student, including Previous Year Papers (PYQ), Proficiency Tests, and Exams. It provides a comprehensive review system with filtering, score visualization, and detailed question-by-question review.

1. Database Schema
Table: paper_test_results
Stores aggregated test submission data.

Column	Type	Nullable	Description
id	uuid	NO	Primary key
student_id	uuid	NO	FK to auth.users
paper_id	uuid	NO	FK to subject_previous_year_papers
subject_id	uuid	YES	FK to subjects
paper_category	text	NO	previous_year / proficiency / exam
score	integer	YES	Number of correct answers
total_questions	integer	NO	Total questions in paper
percentage	numeric	YES	Score percentage
time_taken_seconds	integer	YES	Time spent on test
answers	jsonb	YES	Map of question_id to answer text
grading_status	text	YES	pending / graded / ai_graded
submitted_at	timestamptz	YES	When test was submitted
graded_at	timestamptz	YES	When grading completed
created_at	timestamptz	YES	Record creation time
Table: student_answers
Stores individual question answers (especially for image-based answers).

Column	Type	Nullable	Description
id	uuid	NO	Primary key
user_id	uuid	NO	FK to auth.users
question_id	uuid	NO	FK to questions
paper_id	uuid	NO	FK to subject_previous_year_papers
answer_text	text	YES	Text/typed answer
answer_image_url	text	YES	URL to uploaded image answer
extracted_answer_text	text	YES	AI-extracted text from image
extraction_confidence	text	YES	AI extraction confidence level
submitted_at	timestamptz	YES	When answer was submitted
created_at	timestamptz	YES	Record creation time
2. API Endpoints
Fetch Test Results

GET /rest/v1/paper_test_results?select=*,paper:subject_previous_year_papers(exam_name,year,paper_type)&student_id=eq.{user_id}&order=submitted_at.desc
Optional Filters:

&subject_id=eq.{subject_id} - Filter by subject
