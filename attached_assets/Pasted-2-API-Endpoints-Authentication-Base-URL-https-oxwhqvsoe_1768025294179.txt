5. API #3: Get Playback URL (Edge Function)
This is the main API for video playback. It returns CDN URLs for fast streaming.

Endpoint
POST https://oxwhqvsoelqqsblmqkxx.supabase.co/functions/v1/streaming-api
Request Payload

{
  "action": "get-playback-url",
  "recording_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "quality": "720p"
}
Request Headers
Content-Type: application/json
Authorization: Bearer <user_jwt_token>
apikey: <supabase_anon_key>
Response

interface PlaybackUrlResponse {
  hlsUrl: string | null;           // HLS streaming URL (for adaptive bitrate)
  directUrl: string | null;        // Direct MP4 URL (for simple playback)
  quality: string;                 // Current quality: "360p" | "480p" | "720p" | "1080p" | "original"
  availableQualities: string[];    // All available qualities
  duration: number | null;         // Duration in seconds
  expiresAt: number;               // Unix timestamp when URL expires (1 hour)
  usingCdn: boolean;               // true if CDN is active (fast), false if direct B2 (slow)
}
Example Response (CDN Enabled - Fast)

{
  "hlsUrl": null,
  "directUrl": "https://cdn.yourdomain.com/file/class-recordings/videos/2026/01/recording-abc123.mp4",
  "quality": "original",
  "availableQualities": ["original"],
  "duration": 3600,
  "expiresAt": 1736506800000,
  "usingCdn": true
}
Example Response (No CDN - Slow)

{
  "hlsUrl": null,
  "directUrl": "https://f002.backblazeb2.com/file/class-recordings/videos/2026/01/recording-abc123.mp4?Authorization=xxxxx",
  "quality": "original",
  "availableQualities": ["original"],
  "duration": 3600,

15. Network Quality Detection
The app detects network quality to recommend video quality:


interface NetworkQuality {
  connectionType: 'wifi' | '4g' | '3g' | '2g' | 'unknown';
  effectiveBandwidth: number;    // Mbps
  latency: number;               // ms
  recommendedQuality: '360p' | '480p' | '720p' | '1080p';
  isOnline: boolean;
  isSaveData: boolean;
}

// Quality thresholds (Mbps)
const thresholds = {
  '1080p': 5.0,    // >= 5 Mbps
  '720p': 2.5,     // >= 2.5 Mbps
  '480p': 1.0,     // >= 1 Mbps
  '360p': 0.5      // >= 0.5 Mbps
};
Implementation:

Use navigator.connection API (Web) or native network APIs (Mobile)
Poll every 30 seconds for changes
Listen to online/offline events
Auto-adjust quality based on bandwidth

17. Video Playback Flow
1. User navigates to /watch/{recordingId}
   ↓
2. Fetch recording metadata from Supabase
   ↓
3. Call streaming-api with action: "get-playback-url"
   ↓
4. Receive playback URL (CDN or B2)
   ↓
5. Check for HLS URL:
   - If HLS available → Use hls.js for adaptive streaming
   - If only directUrl → Use native <video> with MP4
   ↓
6. Load saved progress from video_watch_progress table
   ↓
7. Seek to last watched position
   ↓
8. Start playback
   ↓
9. Every 10 seconds: Save progress to database
   ↓
10. On video end: Mark as completed (100%)

Date Presets

type DatePreset = 'all' | 'today' | 'week' | 'month';

const getDateRange = (preset: DatePreset) => {
  const now = new Date();
  switch (preset) {
    case 'today':
      return { from: startOfDay(now), to: endOfDay(now) };
    case 'week':
      return { from: startOfWeek(now), to: endOfDay(now) };
    case 'month':
      return { from: startOfMonth(now), to: endOfDay(now) };
    default:
      return { from: undefined, to: undefined };
  }
};
19. Progress Tracking
Auto-Save Logic

// Save progress every 10 seconds while playing
useEffect(() => {
  if (isPlaying && duration > 0) {
    const interval = setInterval(() => {
      const percent = (currentTime / duration) * 100;
      saveProgress({
        recordingId,
        progressSeconds: Math.floor(currentTime),
        progressPercent: percent,
        completed: percent >= 95
      });
    }, 10000); // 10 seconds
    
    return () => clearInterval(interval);
  }
}, [isPlaying, currentTime, duration]);
Resume Playback

// On video load, seek to saved position
const savedPosition = savedProgress?.progress_seconds || 0;
if (savedPosition > 0) {
  videoElement.currentTime = savedPosition;
}
