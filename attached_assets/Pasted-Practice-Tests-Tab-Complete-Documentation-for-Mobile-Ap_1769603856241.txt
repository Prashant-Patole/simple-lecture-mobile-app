Practice & Tests Tab - Complete Documentation for Mobile App Implementation
Overview
The "Practice & Tests" system in SimpleLecture LMS allows students to take various types of tests:

PYQ (Previous Year Questions) - Historical exam papers
Proficiency Tests - Skill assessment tests
Exam/Mock Tests - Practice exams
DPP (Daily Practice Problems) - Topic-specific practice

API Endpoints (Supabase Queries)
1. Fetch Papers for Subject

// File: src/hooks/usePreviousYearPaperQuestions.ts
supabase
  .from("subject_previous_year_papers")
  .select("*")
  .eq("subject_id", subjectId)
  // Optional filters:
  .eq("topic_id", topicId)     // For topic-level
  .eq("chapter_id", chapterId) // For chapter-level
  .is("topic_id", null)        // For chapter-only mode
  .order("year", { ascending: false })
2. Fetch Questions for a Paper

// File: src/hooks/usePreviousYearPaperQuestions.ts
supabase
  .from("questions")
  .select("*")
  .eq("previous_year_paper_id", paperId)
  .order("created_at", { ascending: true })
3. Fetch Tests (Proficiency/Exam/DPP)

// File: src/hooks/useStudentTests.ts
supabase
  .from("tests")
  .select("*")
  .eq("subject_id", subjectId)
  .eq("is_active", true)
  .in("test_type", ["proficiency"]) // or ["practice", "exam", "mock"]
  .order("created_at", { ascending: false })
4. Fetch Questions for a Test

// File: src/components/learning/PreviousYearPapers.tsx
supabase
  .from("test_questions")
  .select(`
    *,
    question:questions(*)
  `)
  .eq("test_id", testId)
  .order("order_number", { ascending: true })
5. Submit Test Result (Tests Table)

// File: src/hooks/useTestResults.ts
supabase
  .from("test_results")
  .insert({
    test_id: testId,
    student_id: userId,
    subject_id: subjectId,
    test_type: "dpp" | "proficiency" | "mock" | "practice",
    score: correctCount,
    total_questions: totalCount,
    percentage: Math.round((correctCount / totalCount) * 100),
    time_taken_seconds: timeTaken,
    answers: { [questionId]: selectedAnswer, ... },
    grading_status: "graded" | "pending"
  })
6. Submit Paper Test Result (PYQ)

// File: src/hooks/usePaperTestResults.ts
supabase
  .from("paper_test_results")
  .insert({
    paper_id: paperId,
    student_id: userId,
    subject_id: subjectId,
    paper_category: "previous_year" | "proficiency" | "exam",
    score: correctCount,
    total_questions: totalCount,
    percentage: percentageScore,
    time_taken_seconds: timeTaken,
    answers: answersObject,
    grading_status: "graded" | "pending"
  })
7. Fetch Submitted Results

// Test Results (from tests table)
supabase
  .from("test_results")
  .select(`*, test:tests(title, test_type, duration_minutes)`)
  .eq("student_id", userId)
  .order("submitted_at", { ascending: false })

// Paper Test Results (from papers table)
supabase
  .from("paper_test_results")
  .select(`*, paper:subject_previous_year_papers(exam_name, year, paper_type)`)
  .eq("student_id", userId)
  .order("submitted_at", { ascending: false })
8. Upload Answer Image

// File: src/hooks/useStudentAnswers.ts
// Upload to Supabase Storage
supabase.storage
  .from("student-answers")

// Save answer record

    user_id: userId,

Edge Functions
ai-compare-math-answers
Used for AI grading of integer/numerical questions where student answer may be mathematically equivalent but formatted differently.


// Invocation
supabase.functions.invoke('ai-compare-math-answers', {
  body: { 
    items: [
      { id: questionId, user_answer: "2x + 3", correct_answer: "3 + 2x" }
    ]
  }
})

// Response
{
  results: [
    { id: questionId, is_equivalent: true }
  ]
}

Data Types
Question Format

interface PaperQuestion {
  id: string;
  question_text: string;
  question_type: "mcq" | "subjective" | "true_false" | "integer";
  question_format: "single_choice" | "multiple_choice" | "true_false" | "subjective" | "integer";
  options: Record<string, { text: string }>;  // { "A": { text: "..." }, "B": { text: "..." } }
  correct_answer: string;  // "A", "B", "C", "D" or numeric value
  explanation?: string;
  difficulty: string;
  marks: number;

Mobile App Implementation Considerations
1. Test Taking UI
Question navigation (next/previous buttons)
Question palette for jumping between questions
Flag/bookmark questions for review
Timer display (countdown for timed tests)
Answer selection (radio buttons for MCQ, text input for integer)
Image upload support for written answers
2. Offline Support
Cache questions locally before test starts
Store answers locally during test
Sync results when back online
3. Timer Handling
Background timer that continues even if app is minimized
Auto-submit when time expires
Resume test if app crashes (store state)
4. Results Display
Score summary with percentage
Correct/Incorrect/Unanswered breakdown
Question-by-question review with correct answers
Option to clear doubts via AI Assistant
5. Re-attempt Logic
Check submittedTestMap or submittedPaperMap for existing submissions
Display "Submitted" badge with last score
Show "Re-attempt" button instead of "Start"
Feature Flags & Test Type Mapping
Test Type Value	Display Tab	Source Table
"proficiency"	Proficiency	tests
"practice"	Test (if no "dpp" in title)	tests
"mock"	Test	tests
"exam"	Test	tests
"dpp"	DPP	tests
Papers with paper_category = "previous_year"	PYQ	subject_previous_year_papers
Summary
This comprehensive documentation covers all aspects of the Practice & Tests functionality. For mobile implementation, you'll need to:

Implement Supabase client for React Native
Create screens for test list, test-taking, and results
Handle timer logic with background processing
Support both MCQ selection and integer input
Implement image upload for written answers
Store test state locally for crash recovery
Sync results with proper error handling
